# AI Documentary Studio
## Software Requirement Specification
## Volume 02 — Part 03 — Project DNA System

**Version:** 1.0
**Status:** Approved
**Priority:** Critical

---

# 1. OBJECTIVE

A **Channel DNA** defines the permanent identity of a YouTube channel. A **Project DNA** defines the identity of **one specific video**.

Every project must generate its own Project DNA before script generation. No script, prompt, or editing decision may begin until the Project DNA has been created.

---

# 2. WHY PROJECT DNA EXISTS

Same channel. Different videos. Different emotions, pacing, and visuals.

| Video | Topic | Emotion | Visual |
|-------|-------|---------|--------|
| Why Your Brain Hates Silence | Psychology | Curiosity | Brain close-ups |
| How Dopamine Controls Your Life | Neuroscience | Wonder | Neural pathways |
| The Psychology of Loneliness | Psychology | Reflection | Solitary figures |

Project DNA stores those differences.

---

User Idea → Channel DNA → Research → Project DNA Generator → Project DNA

# 7. CORE EMOTION

Every project has **one dominant emotion**. Examples:

`Curiosity | Fear | Wonder | Hope | Suspense | Calm | Mystery | Urgency | Reflection`

The Editing Engine and Motion Engine must follow this emotional direction.

---

# 8. LEARNING OBJECTIVE

Every project answers: *"What should the viewer understand by the end of the video?"*

Example: *"Viewer understands why silence changes brain activity."*

Every later engine uses this objective.

---

# 21. PROJECT INTELLIGENCE PROFILE ⭐

A new layer inside Project DNA. The AI must know not only what to do, but what to **avoid**.

## Avoid List
- Overuse of new images
- Repetitive camera angles
- Weak hooks
- Long static scenes
- Excessive transitions
- Visual clutter

## Preferred Patterns
- Reuse images whenever meaning stays the same
- Insert word-level visuals only for high-impact concepts
- Prefer symbolic visuals over literal ones
- Keep scene rhythm varied
- Maintain emotional continuity

## Risk Detection
During AI generation, detect:
- Viewer attention drop risk
- Story repetition
- Prompt inconsistency
- Character drift
- Visual overload
- Slow pacing

# 22. PROJECT COMPILER

After Project DNA is created, compile it:

```
Project DNA → Validation → Optimization → Compiled Project Profile
```

The compiled profile is used during generation for fast access.

---

# 23. PROJECT VALIDATION

Validate before approval:

| Check | Description |
|-------|-------------|
| Missing Goal | Is there a clear goal? |
| Conflicting Emotions | Do emotions contradict? |
| Invalid Runtime | Is duration realistic? |
| Broken Story Arc | Does the arc make sense? |
| Missing Learning Objective | Is there a learning goal? |
| Incomplete Symbol Library | Are symbols defined? |
| Missing Visual Strategy | Is visual direction set? |

# 26. AI DIRECTOR UPDATE

Replace:

```
Channel DNA → Narrative Planner
```

With:

```
Channel DNA → Project DNA → Project Blueprint → Narrative Planner
```

Every engine must receive both the Channel DNA and the compiled Project DNA before processing.

---

# 27. ACCEPTANCE CRITERIA

The Project DNA System is complete when:

1. Every new video automatically generates its own Project DNA
2. The same Channel DNA can produce videos with different personalities
3. The Prompt Engine, Story Engine, Motion Engine, and Timeline Engine all adapt based on Project DNA
4. The Blueprint is generated and validated before script creation
5. All downstream modules receive a compiled Project Profile
